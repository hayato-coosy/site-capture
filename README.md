# site-capture

Webサイトのスクリーンショット取得・記録を目的としたユーティリティリポジトリです。  
検証・アーカイブ・デザインレビュー用途での利用を想定しています。

---

## 概要

`site-capture` は、指定したURLのWebページをキャプチャし、  
**デザイン確認・変更差分チェック・証跡保存**などに活用するための仕組みをまとめたリポジトリです。

- デザイン検証用のキャプチャ取得
- サイトリニューアル前後の比較
- クライアント共有用の記録素材作成

といった用途を主に想定しています。

---

## 主な用途

- Webデザインのレビュー・アーカイブ
- UI変更前後の差分確認
- 提案資料・ポートフォリオ用の記録
- 社内検証・品質管理（QA）

---

## ディレクトリ構成（例）

```
.
├── captures/        # 取得したスクリーンショット
├── scripts/         # キャプチャ用スクリプト
├── config/          # 設定ファイル
└── README.md
```

※ 実装に応じて構成は柔軟に変更してください。

---

## 使い方（想定）

1. キャプチャ対象のURLを設定
2. スクリプトを実行
3. `captures/` 配下に画像が保存される

※ 詳細な手順は今後追記予定です。
